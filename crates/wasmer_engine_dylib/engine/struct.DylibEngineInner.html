The inner contents of DylibEngine

Fields

compiler: Option<Box<dyn Compiler>>

The compiler

features: Features

The WebAssembly features to use

signatures: SignatureRegistry

The signature registry is used mainly to operate with trampolines performantly.

func_data: Arc<FuncDataRegistry>

The backing storage of VMFuncRefs. This centralized store ensures that 2 functions with the same VMCallerCheckedAnyfunc will have the same VMFuncRef. It also guarantees that the VMFuncRefs stay valid until the engine is dropped.

prefixer: Option<Box<dyn Fn(&[u8]) -> String + Send>>

The prefixer returns the a String to prefix each of the functions in the shared object generated by the DylibEngine, so we can assure no collisions.

is_cross_compiling: bool

Whether the Dylib engine will cross-compile.

linker: Linker

The linker to use.

libraries: Vec<Library>

List of libraries loaded by this engine.
