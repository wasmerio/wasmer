4//! This module contains the bulk of the interesting code performing the translation between
5//! WebAssembly bytecode and Cranelift IR.
6//!
7//! The translation is done in one pass, opcode by opcode. Two main data structures are used during
8//! code translations: the value stack and the control stack. The value stack mimics the execution
9//! of the WebAssembly stack machine: each instruction result is pushed onto the stack and
10//! instruction arguments are popped off the stack. Similarly, when encountering a control flow
11//! block, it is pushed onto the control stack and popped off when encountering the corresponding
12//! `End`.
13//!
14//! Another data structure, the translation state, records information concerning unreachable code
15//! status and about if inserting a return at the end of the function is necessary.
16//!
17//! Some of the WebAssembly instructions need information about the environment for which they
18//! are being translated:
19//!
20//! - the loads and stores need the memory base address;
21//! - the `get_global` and `set_global` instructions depend on how the globals are implemented;
22//! - `memory.size` and `memory.grow` are runtime functions;
23//! - `call_indirect` has to translate the function index into the address of where this
24//!   is;
25//!
26//! That is why `translate_function_body` takes an object having the `WasmRuntime` trait as
27//! argument.
28//!
29//! There is extra complexity associated with translation of 128-bit SIMD instructions.
30//! Wasm only considers there to be a single 128-bit vector type.  But CLIF's type system
31//! distinguishes different lane configurations, so considers 8X16, 16X8, 32X4 and 64X2 to be
32//! different types.  The result is that, in wasm, it's perfectly OK to take the output of (eg)
33//! an `add.16x8` and use that as an operand of a `sub.32x4`, without using any cast.  But when
34//! translated into CLIF, that will cause a verifier error due to the apparent type mismatch.
35//!
36//! This file works around that problem by liberally inserting `bitcast` instructions in many
37//! places -- mostly, before the use of vector values, either as arguments to CLIF instructions
38//! or as block actual parameters.  These are no-op casts which nevertheless have different
39//! input and output types, and are used (mostly) to "convert" 16X8, 32X4 and 64X2-typed vectors
40//! to the "canonical" type, 8X16.  Hence the functions `optionally_bitcast_vector`,
41//! `bitcast_arguments`, `pop*_with_bitcast`, `canonicalise_then_jump`,
42//! `canonicalise_then_br{z,nz}`, `is_non_canonical_v128` and `canonicalise_v128_values`.
43//! Note that the `bitcast*` functions are occasionally used to convert to some type other than
44//! 8X16, but the `canonicalise*` functions always convert to type 8X16.
45//!
46//! Be careful when adding support for new vector instructions.  And when adding new jumps, even
47//! if they are apparently don't have any connection to vectors.  Never generate any kind of
48//! (inter-block) jump directly.  Instead use `canonicalise_then_jump` and
49//! `canonicalise_then_br{z,nz}`.
50//!
51//! The use of bitcasts is ugly and inefficient, but currently unavoidable:
52//!
53//! * they make the logic in this file fragile: miss out a bitcast for any reason, and there is
54//!   the risk of the system failing in the verifier.  At least for debug builds.
55//!
56//! * in the new backends, they potentially interfere with pattern matching on CLIF -- the
57//!   patterns need to take into account the presence of bitcast nodes.
58//!
59//! * in the new backends, they get translated into machine-level vector-register-copy
60//!   instructions, none of which are actually necessary.  We then depend on the register
61//!   allocator to coalesce them all out.
62//!
63//! * they increase the total number of CLIF nodes that have to be processed, hence slowing down
64//!   the compilation pipeline.  Also, the extra coalescing work generates a slowdown.
65//!
66//! A better solution which would avoid all four problems would be to remove the 8X16, 16X8,
67//! 32X4 and 64X2 types from CLIF and instead have a single V128 type.
68//!
69//! For further background see also:
70//!   <https://github.com/bytecodealliance/wasmtime/issues/1147>
71//!     ("Too many raw_bitcasts in SIMD code")
72//!   <https://github.com/bytecodealliance/cranelift/pull/1251>
73//!     ("Add X128 type to represent WebAssembly's V128 type")
74//!   <https://github.com/bytecodealliance/cranelift/pull/1236>
75//!     ("Relax verification to allow I8X16 to act as a default vector type")

102/// Given a `Reachability<T>`, unwrap the inner `T` or, when unreachable, set
103/// `state.reachable = false` and return.
104///
105/// Used in combination with calling `prepare_addr` and `prepare_atomic_addr`
106/// when we can statically determine that a Wasm access will unconditionally
107/// trap.
108macro_rules! unwrap_or_return_unreachable_state {
109    ($state:ident, $value:expr_2021) => {
110        match $value {
111            Reachability::Reachable(x) => x,
112            Reachability::Unreachable => {
113                $state.reachable = false;
114                return Ok(());
115            }
116        }
117    };
118}

122/// Translates wasm operators into Cranelift IR instructions. Returns `true` if it inserted
123/// a return.
124pub fn translate_operator<FE: FuncEnvironment + ?Sized>(
125    module_translation_state: &ModuleTranslationState,
126    op: &Operator,
127    builder: &mut FunctionBuilder,
128    state: &mut FuncTranslationState,
129    environ: &mut FE,
130) -> WasmResult<()> {
131    if !state.reachable {
132        translate_unreachable_operator(module_translation_state, op, builder, state, environ)?;
133        return Ok(());
134    }
135
136    // This big match treats all Wasm code operators.
137    match op {
138        /********************************** Locals ****************************************
139         *  `get_local` and `set_local` are treated as non-SSA variables and will completely
140         *  disappear in the Cranelift Code
141         ***********************************************************************************/
142        Operator::LocalGet { local_index } => {
143            let val = builder.use_var(Variable::from_u32(*local_index));
144            state.push1(val);
145            let label = ValueLabel::from_u32(*local_index);
146            builder.set_val_label(val, label);
147        }
148        Operator::LocalSet { local_index } => {
149            let mut val = state.pop1();
150
151            // Ensure SIMD values are cast to their default Cranelift type, I8x16.
152            let ty = builder.func.dfg.value_type(val);
153            if ty.is_vector() {
154                val = optionally_bitcast_vector(val, I8X16, builder);
155            }
156
157            builder.def_var(Variable::from_u32(*local_index), val);
158            let label = ValueLabel::from_u32(*local_index);
159            builder.set_val_label(val, label);
160        }
161        Operator::LocalTee { local_index } => {
162            let mut val = state.peek1();
163
164            // Ensure SIMD values are cast to their default Cranelift type, I8x16.
165            let ty = builder.func.dfg.value_type(val);
166            if ty.is_vector() {
167                val = optionally_bitcast_vector(val, I8X16, builder);
168            }
169
170            builder.def_var(Variable::from_u32(*local_index), val);
171            let label = ValueLabel::from_u32(*local_index);
172            builder.set_val_label(val, label);
173        }
174        /********************************** Globals ****************************************
175         *  `get_global` and `set_global` are handled by the environment.
176         ***********************************************************************************/
177        Operator::GlobalGet { global_index } => {
178            let val = match state.get_global(builder.func, *global_index, environ)? {
179                GlobalVariable::Const(val) => val,
180                GlobalVariable::Memory { gv, offset, ty } => {
181                    let addr = builder.ins().global_value(environ.pointer_type(), gv);
182                    let mut flags = ir::MemFlags::trusted();
183                    // Put globals in the "table" abstract heap category as well.
184                    flags.set_alias_region(Some(ir::AliasRegion::Table));
185                    builder.ins().load(ty, flags, addr, offset)
186                }
187                GlobalVariable::Custom => environ.translate_custom_global_get(
188                    builder.cursor(),
189                    GlobalIndex::from_u32(*global_index),
190                )?,
191            };
192            state.push1(val);
193        }
194        Operator::GlobalSet { global_index } => {
195            match state.get_global(builder.func, *global_index, environ)? {
196                GlobalVariable::Const(_) => panic!("global #{} is a constant", *global_index),
197                GlobalVariable::Memory { gv, offset, ty } => {
198                    let addr = builder.ins().global_value(environ.pointer_type(), gv);
199                    let mut flags = ir::MemFlags::trusted();
200                    // Put globals in the "table" abstract heap category as well.
201                    flags.set_alias_region(Some(ir::AliasRegion::Table));
202                    let mut val = state.pop1();
203                    // Ensure SIMD values are cast to their default Cranelift type, I8x16.
204                    if ty.is_vector() {
205                        val = optionally_bitcast_vector(val, I8X16, builder);
206                    }
207                    debug_assert_eq!(ty, builder.func.dfg.value_type(val));
208                    builder.ins().store(flags, val, addr, offset);
209                    environ.update_global(builder, *global_index, val);
210                }
211                GlobalVariable::Custom => {
212                    let val = state.pop1();
213                    environ.translate_custom_global_set(
214                        builder.cursor(),
215                        GlobalIndex::from_u32(*global_index),
216                        val,
217                    )?;
218                }
219            }
220        }
221        /********************************* Stack misc ***************************************
222         *  `drop`, `nop`, `unreachable` and `select`.
223         ***********************************************************************************/
224        Operator::Drop => {
225            state.pop1();
226        }
227        Operator::Select => {
228            // we can ignore metadata because extern ref must use TypedSelect
229            let (mut arg1, mut arg2, cond) = state.pop3();
230            if builder.func.dfg.value_type(arg1).is_vector() {
231                arg1 = optionally_bitcast_vector(arg1, I8X16, builder);
232            }
233            if builder.func.dfg.value_type(arg2).is_vector() {
234                arg2 = optionally_bitcast_vector(arg2, I8X16, builder);
235            }
236            state.push1(builder.ins().select(cond, arg1, arg2));
237        }
238        Operator::TypedSelect { ty: _ } => {
239            // We ignore the explicit type parameter as it is only needed for
240            // validation, which we require to have been performed before
241            // translation.
242            let (mut arg1, mut arg2, cond) = state.pop3();
243            if builder.func.dfg.value_type(arg1).is_vector() {
244                arg1 = optionally_bitcast_vector(arg1, I8X16, builder);
245            }
246            if builder.func.dfg.value_type(arg2).is_vector() {
247                arg2 = optionally_bitcast_vector(arg2, I8X16, builder);
248            }
249            state.push1(builder.ins().select(cond, arg1, arg2));
250        }
251        Operator::Nop => {
252            // We do nothing
253        }
254        Operator::Unreachable => {
255            builder.ins().trap(ir::TrapCode::UnreachableCodeReached);
256            state.reachable = false;
257        }
258        /***************************** Control flow blocks **********************************
259         *  When starting a control flow block, we create a new `Block` that will hold the code
260         *  after the block, and we push a frame on the control stack. Depending on the type
261         *  of block, we create a new `Block` for the body of the block with an associated
262         *  jump instruction.
263         *
264         *  The `End` instruction pops the last control frame from the control stack, seals
265         *  the destination block (since `br` instructions targeting it only appear inside the
266         *  block and have already been translated) and modify the value stack to use the
267         *  possible `Block`'s arguments values.
268         ***********************************************************************************/
269        Operator::Block { blockty } => {
270            let (params, results) = module_translation_state.blocktype_params_results(blockty)?;
271            let next = block_with_params(builder, results.iter(), environ)?;
272            state.push_block(next, params.len(), results.len());
273        }
274        Operator::Loop { blockty } => {
275            let (params, results) = module_translation_state.blocktype_params_results(blockty)?;
276            let loop_body = block_with_params(builder, params.iter(), environ)?;
277            let next = block_with_params(builder, results.iter(), environ)?;
278            canonicalise_then_jump(builder, loop_body, state.peekn(params.len()));
279            state.push_loop(loop_body, next, params.len(), results.len());
280
281            // Pop the initial `Block` actuals and replace them with the `Block`'s
282            // params since control flow joins at the top of the loop.
283            state.popn(params.len());
284            state
285                .stack
286                .extend_from_slice(builder.block_params(loop_body));
287
288            builder.switch_to_block(loop_body);
289            environ.translate_loop_header(builder.cursor())?;
290        }
291        Operator::If { blockty } => {
292            let val = state.pop1();
293
294            let next_block = builder.create_block();
295            let (params, results) = module_translation_state.blocktype_params_results(blockty)?;
296            let results: Vec<_> = results.iter().copied().collect();
297            let (destination, else_data) = if params == results {
298                // It is possible there is no `else` block, so we will only
299                // allocate a block for it if/when we find the `else`. For now,
300                // we if the condition isn't true, then we jump directly to the
301                // destination block following the whole `if...end`. If we do end
302                // up discovering an `else`, then we will allocate a block for it
303                // and go back and patch the jump.
304                let destination = block_with_params(builder, results.iter(), environ)?;
305                let branch_inst = canonicalise_brif(
306                    builder,
307                    val,
308                    next_block,
309                    &[],
310                    destination,
311                    state.peekn(params.len()),
312                );
313                (
314                    destination,
315                    ElseData::NoElse {
316                        branch_inst,
317                        placeholder: destination,
318                    },
319                )
320            } else {
321                // The `if` type signature is not valid without an `else` block,
322                // so we eagerly allocate the `else` block here.
323                let destination = block_with_params(builder, results.iter(), environ)?;
324                let else_block = block_with_params(builder, params.iter(), environ)?;
325                canonicalise_brif(
326                    builder,
327                    val,
328                    next_block,
329                    &[],
330                    else_block,
331                    state.peekn(params.len()),
332                );
333                builder.seal_block(else_block);
334                (destination, ElseData::WithElse { else_block })
335            };
336
337            builder.seal_block(next_block); // Only predecessor is the current block.
338            builder.switch_to_block(next_block);
339
340            // Here we append an argument to a Block targeted by an argumentless jump instruction
341            // But in fact there are two cases:
342            // - either the If does not have a Else clause, in that case ty = EmptyBlock
343            //   and we add nothing;
344            // - either the If have an Else clause, in that case the destination of this jump
345            //   instruction will be changed later when we translate the Else operator.
346            state.push_if(
347                destination,
348                else_data,
349                params.len(),
350                results.len(),
351                *blockty,
352            );
353        }
354        Operator::Else => {
355            let i = state.control_stack.len() - 1;
356            match state.control_stack[i] {
357                ControlStackFrame::If {
358                    ref else_data,
359                    head_is_reachable,
360                    ref mut consequent_ends_reachable,
361                    num_return_values,
362                    blocktype,
363                    destination,
364                    ..
365                } => {
366                    // We finished the consequent, so record its final
367                    // reachability state.
368                    debug_assert!(consequent_ends_reachable.is_none());
369                    *consequent_ends_reachable = Some(state.reachable);
370
371                    if head_is_reachable {
372                        // We have a branch from the head of the `if` to the `else`.
373                        state.reachable = true;
374
375                        // Ensure we have a block for the `else` block (it may have
376                        // already been pre-allocated, see `ElseData` for details).
377                        let else_block = match *else_data {
378                            ElseData::NoElse {
379                                branch_inst,
380                                placeholder,
381                            } => {
382                                let (params, _results) = module_translation_state
383                                    .blocktype_params_results(&blocktype)?;
384                                debug_assert_eq!(params.len(), num_return_values);
385                                let else_block =
386                                    block_with_params(builder, params.iter(), environ)?;
387                                canonicalise_then_jump(
388                                    builder,
389                                    destination,
390                                    state.peekn(params.len()),
391                                );
392                                state.popn(params.len());
393
394                                builder.change_jump_destination(
395                                    branch_inst,
396                                    placeholder,
397                                    else_block,
398                                );
399                                builder.seal_block(else_block);
400                                else_block
401                            }
402                            ElseData::WithElse { else_block } => {
403                                canonicalise_then_jump(
404                                    builder,
405                                    destination,
406                                    state.peekn(num_return_values),
407                                );
408                                state.popn(num_return_values);
409                                else_block
410                            }
411                        };
412
413                        // You might be expecting that we push the parameters for this
414                        // `else` block here, something like this:
415                        //
416                        //     state.pushn(&control_stack_frame.params);
417                        //
418                        // We don't do that because they are already on the top of the stack
419                        // for us: we pushed the parameters twice when we saw the initial
420                        // `if` so that we wouldn't have to save the parameters in the
421                        // `ControlStackFrame` as another `Vec` allocation.
422
423                        builder.switch_to_block(else_block);
424
425                        // We don't bother updating the control frame's `ElseData`
426                        // to `WithElse` because nothing else will read it.
427                    }
428                }
429                _ => unreachable!(),
430            }
431        }
432        Operator::End => {
433            let frame = state.control_stack.pop().unwrap();
434            let next_block = frame.following_code();
435            let return_count = frame.num_return_values();
436            let return_args = state.peekn_mut(return_count);
437
438            canonicalise_then_jump(builder, next_block, return_args);
439            // You might expect that if we just finished an `if` block that
440            // didn't have a corresponding `else` block, then we would clean
441            // up our duplicate set of parameters that we pushed earlier
442            // right here. However, we don't have to explicitly do that,
443            // since we truncate the stack back to the original height
444            // below.
445
446            builder.switch_to_block(next_block);
447            builder.seal_block(next_block);
448
449            // If it is a loop we also have to seal the body loop block
450            if let ControlStackFrame::Loop { header, .. } = frame {
451                builder.seal_block(header)
452            }
453
454            frame.truncate_value_stack_to_original_size(&mut state.stack);
455            state
456                .stack
457                .extend_from_slice(builder.block_params(next_block));
458        }
459        /**************************** Branch instructions *********************************
460         * The branch instructions all have as arguments a target nesting level, which
461         * corresponds to how many control stack frames do we have to pop to get the
462         * destination `Block`.
463         *
464         * Once the destination `Block` is found, we sometimes have to declare a certain depth
465         * of the stack unreachable, because some branch instructions are terminator.
466         *
467         * The `br_table` case is much more complicated because Cranelift's `br_table` instruction
468         * does not support jump arguments like all the other branch instructions. That is why, in
469         * the case where we would use jump arguments for every other branch instruction, we
470         * need to split the critical edges leaving the `br_tables` by creating one `Block` per
471         * table destination; the `br_table` will point to these newly created `Blocks` and these
472         * `Block`s contain only a jump instruction pointing to the final destination, this time with
473         * jump arguments.
474         *
475         * This system is also implemented in Cranelift's SSA construction algorithm, because
476         * `use_var` located in a destination `Block` of a `br_table` might trigger the addition
477         * of jump arguments in each predecessor branch instruction, one of which might be a
478         * `br_table`.
479         ***********************************************************************************/
480        Operator::Br { relative_depth } => {
481            let i = state.control_stack.len() - 1 - (*relative_depth as usize);
482            let (return_count, br_destination) = {
483                let frame = &mut state.control_stack[i];
484                // We signal that all the code that follows until the next End is unreachable
485                frame.set_branched_to_exit();
486                let return_count = if frame.is_loop() {
487                    frame.num_param_values()
488                } else {
489                    frame.num_return_values()
490                };
491                (return_count, frame.br_destination())
492            };
493            let destination_args = state.peekn(return_count);
494            canonicalise_then_jump(builder, br_destination, destination_args);
495            state.popn(return_count);
496            state.reachable = false;
497        }
498        Operator::BrIf { relative_depth } => translate_br_if(*relative_depth, builder, state),
499        Operator::BrTable { targets } => {
500            let default = targets.default();
501            let mut min_depth = default;
502            for depth in targets.targets() {
503                let depth = depth.map_err(from_binaryreadererror_wasmerror)?;
504                if depth < min_depth {
505                    min_depth = depth;
506                }
507            }
508            let jump_args_count = {
509                let i = state.control_stack.len() - 1 - (min_depth as usize);
510                let min_depth_frame = &state.control_stack[i];
511                if min_depth_frame.is_loop() {
512                    min_depth_frame.num_param_values()
513                } else {
514                    min_depth_frame.num_return_values()
515                }
516            };
517            let val = state.pop1();
518            let mut data = Vec::with_capacity(targets.len() as usize);
519            if jump_args_count == 0 {
520                // No jump arguments
521                for depth in targets.targets() {
522                    let depth = depth.map_err(from_binaryreadererror_wasmerror)?;
523                    let block = {
524                        let i = state.control_stack.len() - 1 - (depth as usize);
525                        let frame = &mut state.control_stack[i];
526                        frame.set_branched_to_exit();
527                        frame.br_destination()
528                    };
529                    data.push(builder.func.dfg.block_call(block, &[]));
530                }
531                let block = {
532                    let i = state.control_stack.len() - 1 - (default as usize);
533                    let frame = &mut state.control_stack[i];
534                    frame.set_branched_to_exit();
535                    frame.br_destination()
536                };
537                let block = builder.func.dfg.block_call(block, &[]);
538                let jt = builder.create_jump_table(JumpTableData::new(block, &data));
539                builder.ins().br_table(val, jt);
540            } else {
541                // Here we have jump arguments, but Cranelift's br_table doesn't support them
542                // We then proceed to split the edges going out of the br_table
543                let return_count = jump_args_count;
544                let mut dest_block_sequence = vec![];
545                let mut dest_block_map = HashMap::new();
546                for depth in targets.targets() {
547                    let depth = depth.map_err(from_binaryreadererror_wasmerror)?;
548                    let branch_block = match dest_block_map.entry(depth as usize) {
549                        hash_map::Entry::Occupied(entry) => *entry.get(),
550                        hash_map::Entry::Vacant(entry) => {
551                            let block = builder.create_block();
552                            dest_block_sequence.push((depth as usize, block));
553                            *entry.insert(block)
554                        }
555                    };
556                    data.push(builder.func.dfg.block_call(branch_block, &[]));
557                }
558                let default_branch_block = match dest_block_map.entry(default as usize) {
559                    hash_map::Entry::Occupied(entry) => *entry.get(),
560                    hash_map::Entry::Vacant(entry) => {
561                        let block = builder.create_block();
562                        dest_block_sequence.push((default as usize, block));
563                        *entry.insert(block)
564                    }
565                };
566                let default_branch_block = builder.func.dfg.block_call(default_branch_block, &[]);
567                let jt = builder.create_jump_table(JumpTableData::new(default_branch_block, &data));
568                builder.ins().br_table(val, jt);

571                    builder.seal_block(dest_block);
572                    let real_dest_block = {
573                        let i = state.control_stack.len() - 1 - depth;
574                        let frame = &mut state.control_stack[i];
575                        frame.set_branched_to_exit();
576                        frame.br_destination()
577                    };
578                    let destination_args = state.peekn_mut(return_count);
579                    canonicalise_then_jump(builder, real_dest_block, destination_args);
580                }
581                state.popn(return_count);
582            }
583            state.reachable = false;
584        }

587                let frame = &mut state.control_stack[0];
588                frame.num_return_values()

2297/// Deals with a Wasm instruction located in an unreachable portion of the code. Most of them
2298/// are dropped but special ones like `End` or `Else` signal the potential end of the unreachable
2299/// portion so the translation state must be updated accordingly.
2300fn translate_unreachable_operator<FE: FuncEnvironment + ?Sized>(
2301    module_translation_state: &ModuleTranslationState,
2302    op: &Operator,
2303    builder: &mut FunctionBuilder,
2304    state: &mut FuncTranslationState,
2305    environ: &mut FE,
2306) -> WasmResult<()> {
2307    debug_assert!(!state.reachable);
2308    match *op {
2309        Operator::If { blockty } => {
2310            // Push a placeholder control stack entry. The if isn't reachable,
2311            // so we don't have any branches anywhere.
2312            state.push_if(
2313                ir::Block::reserved_value(),
2314                ElseData::NoElse {
2315                    branch_inst: ir::Inst::reserved_value(),
2316                    placeholder: ir::Block::reserved_value(),
2317                },
2318                0,
2319                0,
2320                blockty,
2321            );
2322        }
2323        Operator::Loop { blockty: _ } | Operator::Block { blockty: _ } => {
2324            state.push_block(ir::Block::reserved_value(), 0, 0);
2325        }
2326        Operator::Else => {
2327            let i = state.control_stack.len() - 1;
2328            match state.control_stack[i] {
2329                ControlStackFrame::If {
2330                    ref else_data,
2331                    head_is_reachable,
2332                    ref mut consequent_ends_reachable,
2333                    blocktype,
2334                    ..
2335                } => {
2336                    debug_assert!(consequent_ends_reachable.is_none());
2337                    *consequent_ends_reachable = Some(state.reachable);
2338
2339                    if head_is_reachable {
2340                        // We have a branch from the head of the `if` to the `else`.
2341                        state.reachable = true;
2342
2343                        let else_block = match *else_data {
2344                            ElseData::NoElse {
2345                                branch_inst,
2346                                placeholder,
2347                            } => {
2348                                let (params, _results) = module_translation_state
2349                                    .blocktype_params_results(&blocktype)?;
2350                                let else_block =
2351                                    block_with_params(builder, params.iter(), environ)?;
2352                                let frame = state.control_stack.last().unwrap();
2353                                frame.truncate_value_stack_to_else_params(&mut state.stack);
2354
2355                                // We change the target of the branch instruction.
2356                                builder.change_jump_destination(
2357                                    branch_inst,
2358                                    placeholder,
2359                                    else_block,
2360                                );
2361                                builder.seal_block(else_block);
2362                                else_block
2363                            }
2364                            ElseData::WithElse { else_block } => {
2365                                let frame = state.control_stack.last().unwrap();
2366                                frame.truncate_value_stack_to_else_params(&mut state.stack);
2367                                else_block
2368                            }
2369                        };
2370
2371                        builder.switch_to_block(else_block);
2372
2373                        // Again, no need to push the parameters for the `else`,
2374                        // since we already did when we saw the original `if`. See
2375                        // the comment for translating `Operator::Else` in
2376                        // `translate_operator` for details.
2377                    }
2378                }
2379                _ => unreachable!(),
2380            }
2381        }
2382        Operator::End => {
2383            let stack = &mut state.stack;
2384            let control_stack = &mut state.control_stack;
2385            let frame = control_stack.pop().unwrap();
2386
2387            // Pop unused parameters from stack.
2388            frame.truncate_value_stack_to_original_size(stack);
2389
2390            let reachable_anyway = match frame {
2391                // If it is a loop we also have to seal the body loop block
2392                ControlStackFrame::Loop { header, .. } => {
2393                    builder.seal_block(header);
2394                    // And loops can't have branches to the end.
2395                    false
2396                }
2397                // If we never set `consequent_ends_reachable` then that means
2398                // we are finishing the consequent now, and there was no
2399                // `else`. Whether the following block is reachable depends only
2400                // on if the head was reachable.
2401                ControlStackFrame::If {
2402                    head_is_reachable,
2403                    consequent_ends_reachable: None,
2404                    ..
2405                } => head_is_reachable,
2406                // Since we are only in this function when in unreachable code,
2407                // we know that the alternative just ended unreachable. Whether
2408                // the following block is reachable depends on if the consequent
2409                // ended reachable or not.
2410                ControlStackFrame::If {
2411                    head_is_reachable,
2412                    consequent_ends_reachable: Some(consequent_ends_reachable),
2413                    ..
2414                } => head_is_reachable && consequent_ends_reachable,
2415                // All other control constructs are already handled.
2416                _ => false,
2417            };
2418
2419            if frame.exit_is_branched_to() || reachable_anyway {
2420                builder.switch_to_block(frame.following_code());
2421                builder.seal_block(frame.following_code());
2422
2423                // And add the return values of the block but only if the next block is reachable
2424                // (which corresponds to testing if the stack depth is 1)
2425                stack.extend_from_slice(builder.block_params(frame.following_code()));
2426                state.reachable = true;
2427            }
2428        }
2429        _ => {
2430            // We don't translate because this is unreachable code
2431        }
2432    }
2433
2434    Ok(())
2435}

2437/// This function is a generalized helper for validating that a wasm-supplied
2438/// heap address is in-bounds.
2439///
2440/// This function takes a litany of parameters and requires that the *Wasm*
2441/// address to be verified is at the top of the stack in `state`. This will
2442/// generate necessary IR to validate that the heap address is correctly
2443/// in-bounds, and various parameters are returned describing the valid *native*
2444/// heap address if execution reaches that point.
2445///
2446/// Returns `None` when the Wasm access will unconditionally trap.
2447///
2448/// Returns `(flags, wasm_addr, native_addr)`.
2449fn prepare_addr<FE>(
2450    memarg: &MemArg,
2451    access_size: u8,
2452    builder: &mut FunctionBuilder,
2453    state: &mut FuncTranslationState,
2454    environ: &mut FE,
2455) -> WasmResult<Reachability<(MemFlags, Value, Value)>>
2456where
2457    FE: FuncEnvironment + ?Sized,
2458{
2459    let index = state.pop1();
2460    let heap = state.get_heap(builder.func, memarg.memory, environ)?;
2461
2462    // How exactly the bounds check is performed here and what it's performed
2463    // on is a bit tricky. Generally we want to rely on access violations (e.g.
2464    // segfaults) to generate traps since that means we don't have to bounds
2465    // check anything explicitly.
2466    //
2467    // (1) If we don't have a guard page of unmapped memory, though, then we
2468    // can't rely on this trapping behavior through segfaults. Instead we need
2469    // to bounds-check the entire memory access here which is everything from
2470    // `addr32 + offset` to `addr32 + offset + width` (not inclusive). In this
2471    // scenario our adjusted offset that we're checking is `memarg.offset +
2472    // access_size`. Note that we do saturating arithmetic here to avoid
2473    // overflow. The addition here is in the 64-bit space, which means that
2474    // we'll never overflow for 32-bit wasm but for 64-bit this is an issue. If
2475    // our effective offset is u64::MAX though then it's impossible for for
2476    // that to actually be a valid offset because otherwise the wasm linear
2477    // memory would take all of the host memory!
2478    //
2479    // (2) If we have a guard page, however, then we can perform a further
2480    // optimization of the generated code by only checking multiples of the
2481    // offset-guard size to be more CSE-friendly. Knowing that we have at least
2482    // 1 page of a guard page we're then able to disregard the `width` since we
2483    // know it's always less than one page. Our bounds check will be for the
2484    // first byte which will either succeed and be guaranteed to fault if it's
2485    // actually out of bounds, or the bounds check itself will fail. In any case
2486    // we assert that the width is reasonably small for now so this assumption
2487    // can be adjusted in the future if we get larger widths.
2488    //
2489    // Put another way we can say, where `y < offset_guard_size`:
2490    //
2491    //      n * offset_guard_size + y = offset
2492    //
2493    // We'll then pass `n * offset_guard_size` as the bounds check value. If
2494    // this traps then our `offset` would have trapped anyway. If this check
2495    // passes we know
2496    //
2497    //      addr32 + n * offset_guard_size < bound
2498    //
2499    // which means
2500    //
2501    //      addr32 + n * offset_guard_size + y < bound + offset_guard_size
2502    //
2503    // because `y < offset_guard_size`, which then means:
2504    //
2505    //      addr32 + offset < bound + offset_guard_size
2506    //
2507    // Since we know that that guard size bytes are all unmapped we're
2508    // guaranteed that `offset` and the `width` bytes after it are either
2509    // in-bounds or will hit the guard page, meaning we'll get the desired
2510    // semantics we want.
2511    //
2512    // ---
2513    //
2514    // With all that in mind remember that the goal is to bounds check as few
2515    // things as possible. To facilitate this the "fast path" is expected to be
2516    // hit like so:
2517    //
2518    // * For wasm32, wasmtime defaults to 4gb "static" memories with 2gb guard
2519    //   regions. This means that for all offsets <=2gb, we hit the optimized
2520    //   case for `heap_addr` on static memories 4gb in size in cranelift's
2521    //   legalization of `heap_addr`, eliding the bounds check entirely.
2522    //
2523    // * For wasm64 offsets <=2gb will generate a single `heap_addr`
2524    //   instruction, but at this time all heaps are "dynamic" which means that
2525    //   a single bounds check is forced. Ideally we'd do better here, but
2526    //   that's the current state of affairs.
2527    //
2528    // Basically we assume that most configurations have a guard page and most
2529    // offsets in `memarg` are <=2gb, which means we get the fast path of one
2530    // `heap_addr` instruction plus a hardcoded i32-offset in memory-related
2531    // instructions.
2532    let heap = environ.heaps()[heap].clone();
2533    let addr = match u32::try_from(memarg.offset) {
2534        // If our offset fits within a u32, then we can place the it into the
2535        // offset immediate of the `heap_addr` instruction.
2536        Ok(offset) => bounds_checks::bounds_check_and_compute_addr(
2537            builder,
2538            environ,
2539            &heap,
2540            index,
2541            offset,
2542            access_size,
2543        )?,
2544
2545        // If the offset doesn't fit within a u32, then we can't pass it
2546        // directly into `heap_addr`.
2547        //
2548        // One reasonable question you might ask is "why not?". There's no
2549        // fundamental reason why `heap_addr` *must* take a 32-bit offset. The
2550        // reason this isn't done, though, is that blindly changing the offset
2551        // to a 64-bit offset increases the size of the `InstructionData` enum
2552        // in cranelift by 8 bytes (16 to 24). This can have significant
2553        // performance implications so the conclusion when this was written was
2554        // that we shouldn't do that.
2555        //
2556        // Without the ability to put the whole offset into the `heap_addr`
2557        // instruction we need to fold the offset into the address itself with
2558        // an unsigned addition. In doing so though we need to check for
2559        // overflow because that would mean the address is out-of-bounds (wasm
2560        // bounds checks happen on the effective 33 or 65 bit address once the
2561        // offset is factored in).
2562        //
2563        // Once we have the effective address, offset already folded in, then
2564        // `heap_addr` is used to verify that the address is indeed in-bounds.
2565        //
2566        // Note that this is generating what's likely to be at least two
2567        // branches, one for the overflow and one for the bounds check itself.
2568        // For now though that should hopefully be ok since 4gb+ offsets are
2569        // relatively odd/rare. In the future if needed we can look into
2570        // optimizing this more.
2571        Err(_) => {
2572            let offset = builder.ins().iconst(heap.index_type, memarg.offset as i64);
2573            let adjusted_index =
2574                builder
2575                    .ins()
2576                    .uadd_overflow_trap(index, offset, ir::TrapCode::HeapOutOfBounds);
2577            bounds_checks::bounds_check_and_compute_addr(
2578                builder,
2579                environ,
2580                &heap,
2581                adjusted_index,
2582                0,
2583                access_size,
2584            )?
2585        }
2586    };
2587    let addr = match addr {
2588        Reachability::Unreachable => return Ok(Reachability::Unreachable),
2589        Reachability::Reachable(a) => a,
2590    };
2591
2592    // Note that we don't set `is_aligned` here, even if the load instruction's
2593    // alignment immediate may says it's aligned, because WebAssembly's
2594    // immediate field is just a hint, while Cranelift's aligned flag needs a
2595    // guarantee. WebAssembly memory accesses are always little-endian.
2596    let mut flags = MemFlags::new();
2597    flags.set_endianness(ir::Endianness::Little);
2598
2599    if heap.memory_type.is_some() {
2600        // Proof-carrying code is enabled; check this memory access.
2601        flags.set_checked();
2602    }
2603
2604    // The access occurs to the `heap` disjoint category of abstract
2605    // state. This may allow alias analysis to merge redundant loads,
2606    // etc. when heap accesses occur interleaved with other (table,
2607    // vmctx, stack) accesses.
2608    flags.set_alias_region(Some(ir::AliasRegion::Heap));
2609
2610    Ok(Reachability::Reachable((flags, index, addr)))
2611}

2613fn align_atomic_addr(
2614    memarg: &MemArg,
2615    loaded_bytes: u8,
2616    builder: &mut FunctionBuilder,
2617    state: &mut FuncTranslationState,
2618) {
2619    // Atomic addresses must all be aligned correctly, and for now we check
2620    // alignment before we check out-of-bounds-ness. The order of this check may
2621    // need to be updated depending on the outcome of the official threads
2622    // proposal itself.
2623    //
2624    // Note that with an offset>0 we generate an `iadd_imm` where the result is
2625    // thrown away after the offset check. This may truncate the offset and the
2626    // result may overflow as well, but those conditions won't affect the
2627    // alignment check itself. This can probably be optimized better and we
2628    // should do so in the future as well.
2629    if loaded_bytes > 1 {
2630        let addr = state.pop1(); // "peek" via pop then push
2631        state.push1(addr);
2632        let effective_addr = if memarg.offset == 0 {
2633            addr
2634        } else {
2635            builder
2636                .ins()
2637                .iadd_imm(addr, i64::from(memarg.offset as i32))
2638        };
2639        debug_assert!(loaded_bytes.is_power_of_two());
2640        let misalignment = builder
2641            .ins()
2642            .band_imm(effective_addr, i64::from(loaded_bytes - 1));
2643        let f = builder.ins().icmp_imm(IntCC::NotEqual, misalignment, 0);
2644        builder.ins().trapnz(f, ir::TrapCode::HeapMisaligned);
2645    }
2646}

2678/// Translate a load instruction.
2679///
2680/// Returns the execution state's reachability after the load is translated.
2681fn translate_load<FE: FuncEnvironment + ?Sized>(
2682    memarg: &MemArg,
2683    opcode: ir::Opcode,
2684    result_ty: Type,
2685    builder: &mut FunctionBuilder,
2686    state: &mut FuncTranslationState,
2687    environ: &mut FE,
2688) -> WasmResult<Reachability<()>> {
2689    let mem_op_size = mem_op_size(opcode, result_ty);
2690    let (flags, wasm_index, base) =
2691        match prepare_addr(memarg, mem_op_size, builder, state, environ)? {
2692            Reachability::Unreachable => return Ok(Reachability::Unreachable),
2693            Reachability::Reachable((f, i, b)) => (f, i, b),
2694        };
2695
2696    environ.before_load(builder, mem_op_size, wasm_index, memarg.offset);
2697
2698    let (load, dfg) = builder
2699        .ins()
2700        .Load(opcode, result_ty, flags, Offset32::new(0), base);
2701    state.push1(dfg.first_result(load));
2702    Ok(Reachability::Reachable(()))
2703}

2746fn fold_atomic_mem_addr(
2747    linear_mem_addr: Value,
2748    memarg: &MemArg,
2749    access_ty: Type,
2750    builder: &mut FunctionBuilder,
2751) -> Value {
2752    let access_ty_bytes = access_ty.bytes();
2753    let final_lma = if memarg.offset > 0 {
2754        assert!(builder.func.dfg.value_type(linear_mem_addr) == I32);
2755        let linear_mem_addr = builder.ins().uextend(I64, linear_mem_addr);
2756        let a = builder
2757            .ins()
2758            .iadd_imm(linear_mem_addr, memarg.offset as i64);
2759        let r = builder
2760            .ins()
2761            .icmp_imm(IntCC::UnsignedGreaterThanOrEqual, a, 0x1_0000_0000i64);
2762        builder.ins().trapnz(r, ir::TrapCode::HeapOutOfBounds);
2763        builder.ins().ireduce(I32, a)
2764    } else {
2765        linear_mem_addr
2766    };
2767    assert!(access_ty_bytes == 4 || access_ty_bytes == 8);
2768    let final_lma_misalignment = builder
2769        .ins()
2770        .band_imm(final_lma, i64::from(access_ty_bytes - 1));
2771    let f = builder
2772        .ins()
2773        .icmp_imm(IntCC::Equal, final_lma_misalignment, i64::from(0));
2774    builder.ins().trapz(f, ir::TrapCode::HeapMisaligned);
2775    final_lma

3257/// Cast to I8X16, any vector values in `values` that are of "non-canonical" type (meaning, not
3258/// I8X16), and return them in a slice.  A pre-scan is made to determine whether any casts are
3259/// actually necessary, and if not, the original slice is returned.  Otherwise the cast values
3260/// are returned in a slice that belongs to the caller-supplied `SmallVec`.
3261fn canonicalise_v128_values<'a>(
3262    tmp_canonicalised: &'a mut SmallVec<[ir::Value; 16]>,
3263    builder: &mut FunctionBuilder,
3264    values: &'a [ir::Value],
3265) -> &'a [ir::Value] {
3266    debug_assert!(tmp_canonicalised.is_empty());
3267    // First figure out if any of the parameters need to be cast.  Mostly they don't need to be.
3268    let any_non_canonical = values
3269        .iter()
3270        .any(|v| is_non_canonical_v128(builder.func.dfg.value_type(*v)));
3271    // Hopefully we take this exit most of the time, hence doing no heap allocation.
3272    if !any_non_canonical {
3273        return values;
3274    }
3275    // Otherwise we'll have to cast, and push the resulting `Value`s into `canonicalised`.
3276    for v in values {
3277        tmp_canonicalised.push(if is_non_canonical_v128(builder.func.dfg.value_type(*v)) {
3278            builder.ins().bitcast(
3279                I8X16,
3280                MemFlags::new().with_endianness(ir::Endianness::Little),
3281                *v,
3282            )
3283        } else {
3284            *v
3285        });
3286    }
3287    tmp_canonicalised.as_slice()
3288}

3291/// don't have that type.  This is done in somewhat roundabout way so as to ensure that we
3292/// almost never have to do any heap allocation.
3293fn canonicalise_then_jump(
3294    builder: &mut FunctionBuilder,

3296    params: &[ir::Value],

3352pub fn bitcast_arguments<'a>(
3353    builder: &FunctionBuilder,
3354    arguments: &'a mut [Value],
3355    params: &[ir::AbiParam],
3356    param_predicate: impl Fn(usize) -> bool,
3357) -> Vec<(Type, &'a mut Value)> {
3358    let filtered_param_types = params
3359        .iter()
3360        .enumerate()
3361        .filter(|(i, _)| param_predicate(*i))
3362        .map(|(_, param)| param.value_type);
3363
3364    // zip_eq, from the itertools::Itertools trait, is like Iterator::zip but panics if one
3365    // iterator ends before the other. The `param_predicate` is required to select exactly as many
3366    // elements of `params` as there are elements in `arguments`.
3367    let pairs = filtered_param_types.zip_eq(arguments.iter_mut());
3368
3369    // The arguments which need to be bitcasted are those which have some vector type but the type
3370    // expected by the parameter is not the same vector type as that of the provided argument.
3371    pairs
3372        .filter(|(param_type, _)| param_type.is_vector())
3373        .filter(|(param_type, arg)| {
3374            let arg_type = builder.func.dfg.value_type(**arg);
3375            assert!(
3376                arg_type.is_vector(),
3377                "unexpected type mismatch: expected {}, argument {} was actually of type {}",
3378                param_type,
3379                *arg,
3380                arg_type
3381            );
3382
3383            // This is the same check that would be done by `optionally_bitcast_vector`, except we
3384            // can't take a mutable borrow of the FunctionBuilder here, so we defer inserting the
3385            // bitcast instruction to the caller.
3386            arg_type != *param_type
3387        })
3388        .collect()
3389}
