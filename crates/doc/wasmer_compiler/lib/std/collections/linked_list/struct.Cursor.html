A cursor over a LinkedList.

A Cursor is like an iterator, except that it can freely seek back-and-forth.

Cursors always rest between two elements in the list, and index in a logically circular way. To accommodate this, there is a â€œghostâ€ non-element that yields None between the head and tail of the list.

When created, cursors start at the front of the list, or the â€œghostâ€ non-element if the list is empty.
